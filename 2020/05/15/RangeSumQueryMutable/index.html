    Question: 307 – RangeSumQueryMutable

    Given a int array nums, find the summary of elements between indicies i and j;
    The update (i, val) function modifies nums by updating the element at index i to val.

    Example1
    Given nums = [1, 3, 5]

    sumRange(0, 2) -> 9
    update(1, 2)
    sumRange(0, 2) -> 8

    Note
    The array is only modifiable by the update function.
    You may assume the number of calls to update and sumRange function is distributed evenly.

    Analysis
    I the extra array to store other infomation than nums, the sumRange(i, j) will cost O(n) time to proccess, and update method will cost only O(1) time to process.

    But if we use accum[] array to store the all the previous sum, if will cost O(1) to get and O(n) to update.

    Using linear accumulation method cannot be optimized to less time complexity since the linear is the O(n) complexity

    Thus, can use segment tree to do this problem. Segment tree just store the sum as segment or part sum. The update and get time complexity are both O(logn).

    Result
    Runtime: 9 ms, faster than 84.85% of Java online submissions for Range Sum Query - Mutable.
    Memory Usage: 45.5 MB, less than 87.50% of Java online submissions for Range Sum Query - Mutable.
